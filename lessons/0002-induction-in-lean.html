Lesson 2 · The induction tactic

Induction in Lean

You proved facts about ℕ by induction long before you met Lean. This lesson is only about how to hand that argument to the machine — and one surprise about which half of a proof is “free.”

In Lesson 1 you learned that a Lean proof is a goal you steer to No goals with tactics, and you met rfl and rw. Those two close finite goals. But almost everything in Tao Chapter 2 — n + 0 = n, commutativity, associativity — is a statement about all natural numbers, and you proved them by Tao’s Axiom 2.5, the principle of induction. Lean has one tactic for exactly that.

The one idea. Induction in Lean is: name the variable, and Lean hands you back two goals — a base case (the statement at 0) and a successor case that already has the induction hypothesis sitting in your context. You close each with the tactics you already know. You never build the chain of implications by hand; Lean stitches the base and step together for you.1

That is Axiom 2.5 turned into a machine. Tao says: if P(0) holds and P(n) ⟹ P(n++) for every n, then P holds everywhere. The induction tactic gives you precisely those two obligations and nothing else.

The companion speaks two dialects — recognise both

The teorth/analysis companion ships Axiom 2.5 as an ordinary theorem, Nat.induction,2 so you invoke it with apply. Mainstream Lean has a built-in induction tactic. You’ll see both; they deliver the same two goals.

Companion house style

-- goal: n + 0 = n
revert n; apply induction
. -- base: ⊢ 0 + 0 = 0
  exact zero_add 0
intro n ih  -- step
-- ih : n + 0 = n
-- ⊢ n++ + 0 = n++

Mainstream Lean / Mathlib

-- goal: ∀ n, P n
induction n with
| zero =>
    -- ⊢ P 0
    ..
| succ k ih =>
    -- ih : P k
    -- ⊢ P (k++)

Two things to internalise. First, revert n turns your goal P n (where n is a fixed local) back into ∀ n, P n so that apply induction matches — and, in general, reverting any variable that still mentions n is what keeps your induction hypothesis strong enough to use.3 Second, the name ih is yours, not a keyword: it is just what you typed to name the hypothesis Tao calls “whenever P(n) is true.”

The surprise: which side is free?

Here is the thing that trips up every mathematician arriving in Lean. On paper, 0 + n = n and n + 0 = n feel equally trivial. In Lean, one of them is free and the other needs a full induction — and which is which depends entirely on how addition was defined.

The companion follows Tao: 0 + m := m and (n++) + m := (n+m)++, recursing on the left. So 0 + m = m is literally the definition unfolding (rfl), while n + 0 = n leaves the recursed variable n unresolved and must be proved by induction.4 The Natural Number Game does the mirror image, so there it’s n + 0 = n that’s free. Same two facts, opposite difficulty.

Don’t memorise “zero_add is easy.” Memorise: the recursed argument is the easy side.

Watch a real induction run

This is the companion’s actual proof of Lemma 2.2.2, n + 0 = n (Nat.add_zero). Follow the tactic state, exactly as in Lesson 1.

Goal
n : Nat  ⊢  n + 0 = n
▼  revert n; apply induction
Two goals — base, then step
⊢ 0 + 0 = 0 ⊢ ∀ n, n + 0 = n → n++ + 0 = n++
▼  . exact zero_add 0  (base: 0+0=0 is zero_add, free by rfl)
Base closed — step remains
⊢ ∀ n, n + 0 = n → n++ + 0 = n++
▼  intro n ih
Successor case — note the hypothesis
n : Nat,  ih : n + 0 = n  ⊢  n++ + 0 = n++
▼  calc (n++) + 0 = (n + 0)++ := by rfl  (succ_add is free)
▼  _ = n++ := by rw [ih]
Tactic state
No goals

The whole proof is: split with induction, close the base with the free lemma, then in the step rewrite with the induction hypothesis ih. That shape repeats for commutativity, associativity, everything.

Now do it for real (≈20 min)

The tightest feedback loop for induction is still the Natural Number Game. You finished Tutorial World (just rfl and rw); Addition World is where induction first appears — and its very first level is the inductive direction.

  1. Open the game and enter Addition World, Level 1. Goal: 0 + n = n (this is zero_add). Recall: in NNG addition recurses on the second argument, so this is the side that needs induction.
  2. Type induction n with d hd. You get two goals: base 0 + 0 = 0, and a successor goal with hd : 0 + d = d where you must prove 0 + succ d = succ d. (NNG writes the successor as succ d, not d++ — that ++ is the companion’s notation.)
  3. Close the base with rw [add_zero]. In the step, rw [add_succ] turns 0 + succ d into succ (0 + d), then rw [hd] finishes — watch the goal collapse to succ d = succ d.
  4. Keep going to the Addition World boss, add_comm. (Tip the game gives you: induct on the right variable, or you’ll strand a + b you can’t simplify.)
